## 🧱 The AWS-EM capability pillars (the roadmap to "ultimate")

### Pillar 1 — Find & qualify AWS-fit accounts
*Real EM:* builds the net-new target list and sizes the market. *AWS truth #5:* AWS funds **net-new consumption** (MAP/SPI for greenfield/migration/modernization; "$543B incremental, 90%+ workloads not yet on cloud").
- **Today:** multi-cloud Discovery agent (AWS/GCP/Azure, daily cron); free `cloud-detect`/`aws-detect`; ICP scoring; SCB market-sizing (TAM/ICP); innovation-maturity bands (0–4, grounded); BuiltWith tech stack.
- **Ultimate:** sharpen **MAP-candidate** detection — flag *not-on-AWS / on-prem / other-cloud* accounts as net-new (= funded) and auto-classify **migrate vs modernize vs greenfield**; lead the worklist with net-new, not renewals.

### Pillar 2 — Shape co-sell-ready opportunities (the ACE checklist)  ← biggest new lever
*Real EM:* gets the opportunity to pass AWS validation. *AWS truths #1–3:* an opp is "validated" only through **Submitted → In-review → Approved** (≤5 biz days; incomplete → "action required"); required payload = **customer business problem, use case, future close date, "Co-Sell with AWS" + named partner needs, est. MRR**, at stage **Qualified+**; judged on deal size / solution alignment / engagement status.
- **Today:** CoPilotPanel = funding brief + AWS-MEDDIC qual + co-sell prep + outreach (partial, prose-only).
- **Ultimate:** an **"ACE-ready scorecard"** on every opportunity — Smith scores against the exact AWS rubric, flags the gap (*"5/6 — missing business problem"*), **drafts each missing field**, and (Pillar 6, via pc-mcp) submits to ACE. Most automatable + most defensible thing Smith can do.

### Pillar 3 — Build the migration business case (MAP Assess)  ← Cloudamize
*Real EM:* produces the TCO, right-size, 7-R disposition, run-rate. *AWS truth #4:* **MAP 2.0**, phases **Assess → Mobilize → Migrate & Modernize**; credits tied to real consumption via the `map-migrated` tag.
- **Today:** real **AWS pricing** in the TCO (`aws-pricing`, official Price List API, SigV4); migration assessment kit; Smith flags *"est. run-rate ~$Xk/mo (validate)."*
- **Ultimate:** **Cloudamize integration** = the "validate" (push account → pull TCO/right-size/7-R/run-rate → autofill the ACE MRR + MAP migration fields). Until then, deepen the in-house Assess kit on real pricing. (OLA = a free pre-migration cost wedge to surface early.)

### Pillar 4 — Map funding + track partner eligibility
*Real EM:* matches each deal to the right program and knows what the partner can unlock. *AWS truths #6–7:* unlock chain **Partner Path → free FTR (3 yrs) → Validated Stage → ACE → Specialization**; funding-request eligibility = **Partner Path/Stage + MDF wallet**; programs = MAP/POA(PoC)/MDF/credits/OLA.
- **Today:** funding-eligibility engine (time-series + eligibility, report-only); `FUNDING_REF` in grounding.
- **Ultimate:** map each fundable deal → **specific program + its gate**; track the *partner's own* readiness state (FTR? Validated? Specialization? wallet?) to show what's actually unlockable; **honesty guardrail** — don't quote "up to 25% MAP credits" (partner-marketing, not AWS-owned) or unverified GenAI/Well-Architected *funding* names.

### Pillar 5 — Open decision-makers + draft outreach
*Real EM:* the right person, at the right time. *AWS truth #3:* submit/engage **at Qualified+** (the timing gate).
- **Today:** free `findPeople` (web) decision-maker finder; contacts attach + dedup; outreach draft; **Tier-1** "Open in Gmail / Add to Calendar" (rep's own, prefilled, zero-OAuth).
- **Ultimate:** sharper targeting for migrations (the **infra/cloud-budget owner**, not just CTO); trigger/timing nudges ("now at Qualified → open the deal / submit ACE"); **Tier-2** native send (CASA-gated, expansion).
